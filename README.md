Un motor de juego de Piedra, Papel o Tijera matemáticamente transparente, gobernado por la entropía física del hardware y aritmética de módulo 3. Diseñado para ser 100% inclusivo, seguro y de alto rendimiento.

---

## 🕹️ Modos de Juego y Filosofía de UX

### 🟢 Modo Básico: Interfaz Invisible (Niños y Adultos Mayores)
Elimina la fricción de los teclados móviles complejos. El usuario interactúa de forma nativa a través de cualquier aplicación de mensajería usando marcadores universales:
- **Emojis directos:** `⭐`, `🔴`, `🎮` (Accesibles desde el teclado principal).
- **Caracteres simples:** Signos de puntuación como guiones `-` o más `+`.

### 🔵 Modo Normal: Multiverso Concurrente (Multi-Personaje)
Permite batallas masivas dentro de una misma sala de chat o hilo de texto. El sistema segmenta el párrafo usando las diferentes instancias del marcador para alimentar dinámicamente los flujos de los personajes secundarios.

### 🔴 Modo Avanzado: Componente de Sistema Integrado
El código es totalmente desacoplado y modular, ideal para ser incrustado como un Easter Egg interactivo en pantallas de carga de aplicaciones pesadas o como un sistema de entretenimiento local (Offline) cuando se pierde la conexión de red.

---

## 🛡️ Arquitectura de Optimización y Protección de Hardware

Para garantizar que el juego corra de manera fluida en cualquier dispositivo (desde smartphones antiguos hasta consolas o servidores en la nube), el motor cuenta con dos escudos algorítmicos integrados:

1. **Reciclaje de Hilos de Tiempo (Aritmética Cíclica - Round-Robin):**
   Si una entrada de texto convoca a una cantidad masiva de personajes (ej: 20 PJs), el script **no estresa la CPU** abriendo 20 subprocesos de tiempo. En su lugar, mide únicamente la latencia física de los primeros fragmentos de texto (máximo 3) y distribuye los resultados entre todo el multiverso usando un mapeo cíclico indexado (`Posición % Nodos_Calculados`).

2. **Escudo Antidesbordamiento (Safe Reset Contingency):**
   Si el sistema detecta una entrada maliciosa o abusiva con 4 o más instancias del marcador, activa de inmediato un protocolo de mitigación de bajo consumo. Interrumpe el análisis por expresiones regulares para proteger la memoria RAM y resuelve el combate instantáneamente usando una matriz de números pseudoaleatorios aislados, manteniendo la estabilidad absoluta del hardware.
